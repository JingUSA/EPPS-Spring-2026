  1. What is the problem? Why?

The main problem is that the dependent variable is not truly continuous. It is categorical and has only a small number of categories. In this case, votetsai is essentially a binary outcome, indicating whether the respondent voted for Tsai or not. Because linear regression assumes a continuous dependent variable, OLS is not the best model here. The fitted values can fall outside the 0 to 1 range, and the linearity and constant-variance assumptions are not well suited for a binary dependent variable.

This is also consistent with the workshop code, which later recodes the vote variable into a two-category factor and uses logistic regression for prediction rather than ordinary least squares.

  1. What can be done to improve prediction of the dependent variable?

A better approach is to use a classification model instead of linear regression. The most natural improvement is logistic regression, because the dependent variable is binary. Logistic regression constrains predicted probabilities to the 0 to 1 range and is more appropriate for modeling vote choice.

For example:

logit_model <- glm(votetsai ~ age + edu + income,
                   data = teds_ml,

The workshop code later includes variables such as gender, Taiwanese identity, economic evaluations, and party-related measures, and then compares logistic regression, decision tree, and random forest models. Brief conclusion

Using lm() here is helpful as an exercise, but it is not ideal because the dependent variable is categorical. A logistic regression model is more appropriate, and prediction can be further improved by adding relevant political variables and using classification methods.
